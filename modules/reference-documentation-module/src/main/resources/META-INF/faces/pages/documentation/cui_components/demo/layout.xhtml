Custom Sizing

Resize your viewport to see the impact on the widths of the columns.

In general grid classes (col-xx-yy) apply to devices with screen widths greater than or equal to the breakpoint.

Thus, if a grid class is not present it will use the size/width of the next smaller grid class.

If no smaller grid class is present it will default to the size 12.

"col-md-4" equivalent to "col-xs-12 col-sm-12 col-md-4 col-lg-4" "col-sm-6 col-lg-4" equivalent to "col-xs-12 col-sm-6 col-md-6 col-lg-4" | col-lg-3 col-md-4 col-sm-6 col-xs defaults to 12 (if not present) | | col-lg-3 col-md-4 col-sm-6 col-xs defaults to 12 (if not present) | | col-lg-2 col-md-4 col-sm-6 col-xs-6 | | col-lg is not present --> it defaults to col-md- col-md-4 col-sm-6 col-xs-6 |

boot:navigationMenu

Navigation menu item demo. The menu item is set to an active by adding the current view ID to the components model in the backing bean. The component renderer sets a 'data-item-active' HTML attribute for active components. By defining a CSS rule like 'li[data-item-active] {...}' the 'active' styling can be changed accordingly.
