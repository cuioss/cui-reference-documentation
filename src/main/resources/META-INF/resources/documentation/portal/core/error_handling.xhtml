Error Handling

Overview

The portal supportes assembling a web page of different sections, that can handle errors independently. For example when displaying a dashboard, some sections may not be available due to problems in the communication or in the backend itself. These sections should only display an error message but will not show any content. On the other hand other sections may work properly and should be displayed intact.

There exists two different types of templates for this kind of error handling sections: One which is using a lazy loading concept, where loading of the actual content does not block loading of the page itself, and one without lazy loading.

In both ways the error handling bases upon the ResultObject.
